Detailed Description

Processes command arguments.

Commands can request parameters by name and have them parsed into the expected object type at the same time. If the parameter has no provided value, cannot be parsed or required conditions for parsing the object are not met, an InvalidArgumentException is thrown and should be caught by the command dispatcher.

Other exceptions thrown are DuplicateArgumentException, InvalidParameterException and TooManyArgsException.

ItemStack [] getItemStack ( @Nullable CommandSender  sender,
String  parameterName 
) throws InvalidArgumentException

Gets an argument as an ItemStack[].

The supplied argument can be a parsable string representing an ItemStack

The supplied argument can also be "inhand" for the stack in the players hand, "inventory" to return all items in the players inventory, or "hotbar" to return all items in the players hotbar. All items returned from the player are cloned objects.

Use getString(parameterName) method on the same parameter to determine if the player typed "inhand", "chest", or "hotbar" if that information is needed.

If the command sender is not a player, and therefore has no chest, the argument will only be valid if a parsable item stack string was provided.

String  parameterName,
ILocationHandler  locationHandler 
) throws InvalidArgumentException

Gets an argument as a location.

Possible values are "current" or "select"

If the argument value is "current", the players current location is returned via the ILocationHandler.

If the argument value is "select", the player is asked to click on the location to be selected and the value is return via the ILocationHandler.

If the CommandSender is not a player, the argument is always considered invalid.
