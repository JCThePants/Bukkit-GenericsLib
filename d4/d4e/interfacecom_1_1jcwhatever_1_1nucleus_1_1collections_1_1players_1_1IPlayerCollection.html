Detailed Description

An interface that must be implemented in order to use a collection with a PlayerCollectionTracker, which aids in automatically removing player entries.

The player collection implementation is responsible for instantiating a new PlayerCollectionTracker for itself to use. It also responsible for calling PlayerCollectionTracker#notifyPlayerRemoved method when a player is removed from the collection and PlayerCollectionTracker#notifyPlayerAdded when a player is added to the collection.

The method removePlayer should not call PlayerCollectionTracker#notifyPlayerRemoved or PlayerCollectionTracker#notifyPlayerAdded because it's used to remove the player when the player logs out.

The collection should by synchronized since the removePlayer method may be called from an asynchronous thread.
